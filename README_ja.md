# kakushi 隠し

**ChatGPT・Claude・Gemini に送る前の、日本語ビジネステキスト用 PII マスキング。** 個人名、社名、電話番号、メールアドレス、金額、マイナンバーを自動でマスクして、LLM の返答で原文に戻します。

[ブラウザでデモを試す](https://huggingface.co/spaces/choudai/kakushi) · [English](README.md)

---

## なぜ必要か

社内のメール、議事録、見積書を ChatGPT に貼り付けたい。要約させたい、丁寧な文に整えさせたい、Q&A 用に整理したい。でも、お客様の名前 (`田中太郎`)、取引先名 (`株式会社山田商事`)、金額 (`¥1,500,000`)、連絡先 — それらが OpenAI のサーバーに渡るのは怖い。

kakushi は、その間に挟む薄いレイヤーです。

```python
from kakushi import mask, unmask

masked, mapping = mask(
    "田中太郎様、株式会社山田商事の件で¥1,500,000の見積書を tanaka@example.co.jp にお送りします",
    dictionary={"株式会社山田商事": "取引先A"},   # 任意・取引先名を固定マスキング
)
# masked  → "<PERSON_1>様、取引先Aの件で<AMOUNT_1>の見積書を <EMAIL_1> にお送りします"
# mapping → メモリ上のみ。ディスクには書きません。

reply = call_openai(masked)                   # LLM 呼び出し
final = unmask(reply, mapping)                # プレースホルダ → 原文に復元
```

## インストール

```bash
pip install kakushi          # 正規表現レイヤーのみ。軽量
pip install kakushi[ner]     # + GiNZA 日本語 NER (人名 / 組織 / 地名)
```

## マスキングの 3 層構造

優先度の低い順に、

1. **GiNZA NER** — 日本語 spaCy パイプラインから `PERSON` / `ORG` / `LOCATION` を抽出。
2. **正規表現** — 電話番号、郵便番号、メール、マイナンバー、円表記、IPv4、運転免許、クレジットカード、URL、契約/注文番号、**さらに `株式会社` / `合同会社` / `(株)` を含む日本語社名**。
3. **会社辞書 (任意)** — `{"株式会社山田商事": "取引先A"}`。最優先。値を `None` にすると自動で `<COMPANY_DICT_N>` 番号を振ります。

スパンが重なった場合は `辞書 > 正規表現 > NER` の優先順位、同優先度では長いマッチが採用されます。

## 合成コーパスでの検出率 (100 件、NER オン)

| エンティティ | 検出率 |
|---|---|
| AMOUNT (金額) | **100%** |
| EMAIL | **100%** |
| PERSON (人名) | **100%** |
| PHONE (電話) | **100%** |
| ORG (社名) | **92%** |
| LOCATION (地名) | 60.9% |
| **全体** | **95.7%** |

`python scripts/eval.py --n 100` で再現できます。

## 既知の限界

- **ひらがな始まりの社名** (`株式会社さくら通信`, `合同会社みどり`) は、正規表現で取れません。本文文字にひらがなを含めると助詞境界が崩れるためです。会社辞書に登録してください。
- **複合地名** (`京都府京都市` など) は GiNZA が分割することがあります。
- **マイナンバーのチェックサム検証**はまだ実装していません。12 桁数字のパターンは全て候補とみなします。
- `mask()` が返す `mapping` は **メモリ上のみ**。ログに出したり、ディスクに書いたりしないでください。元の機密情報そのものが入っています。
